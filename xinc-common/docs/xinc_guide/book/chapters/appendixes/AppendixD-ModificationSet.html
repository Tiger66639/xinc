Appendix D: ModificationSet

This appendix contains a reference of all modificationset tasks, i.e. all tasks that can trigger a build to happen.

<buildalways/>

The buildalways task fakes a modification and triggers a build each time Xinc checks for a modification.

Parent Tags

  • <modificationset/>
<project name="ProjectName">
    <trigger>
        <schedule interval="240"/>
    </trigger>
...
    <modificationset>
        <buildalways/>
    </modificationset>
...
</project>

The above example will execute a build every 240 seconds, since the buildalways modificationset will always detect a modification.

Parameters

Name Type Description Default Required
None

 

<svn/>

The svn task checks repository changes between a local working copy and a remote subversion repository.
If a revision difference between the local working copy and the remote repository has been detected a build is triggered.

Parent Tags

  • <modificationset/>
<project name="ProjectName">
    <trigger>
        <schedule interval="240"/>
    </trigger>
...
    <modificationset>
        <svn directory="/var/xinc/projects/ProjectName" update="true"/>
    </modificationset>
...
</project>

The above example will check the local working copy "/var/xinc/projects/ProjectName" every 240 seconds for changes. If a change is detected it will update the local working copy and trigger a build.

Parameters

Name Type Description Default Required
directory String Directory of the local SVN working copy n/a Yes
repository String The remote repository to check/update from n/a Yes
username String Username for the remote subversion repository n/a No
password String Password for the remote subversion repository n/a No
update boolean Determines whether the local working copy should be updated if a change is detected or not. false No
trustServerCert boolean Determines whether the trust-server-cert parameter on svn command should be used or not. false No
