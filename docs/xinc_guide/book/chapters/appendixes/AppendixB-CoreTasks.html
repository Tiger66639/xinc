Appendix B: Core Tasks

This appendix contains a reference of all core tasks, i.e. all tasks that are needed to build a basic project.

<xinc/>

The Root Element Xinc contains one or many <project/> elements.

Supported Nested Tags

  • <project/>

 

<project/>

The Project Element contains all tasks that need to be run for the build process.

Parent Tags

  • <xinc/>
<project name="ProjectName">
<configuration>
    <setting name="loglevel" value="0"/>
</configuration>
...
</project>

Supported Nested Tags

  • <configuration/>
  • <property/>
  • <schedule/>
  • <cron/>
  • <modificationset/>
  • <builders/>
  • <publishers/>

 

<configuration/>

The Configuration Task lets you specify some project specific settings.

Parent Tags

  • <project/>
<configuration>
    <setting name="loglevel" value="0"/>
</configuration>

Parameters

None

Supported Nested Tags

  • <setting/>

 

<setting/>

The Setting Task is a nested element of the configuration task and lets you specify name value pairs.

<configuration>
    <setting name="loglevel" value="0"/>
    <setting name="timezone" value="Europe/Berlin"/>
</configuration>

Parent Tags

  • <configuration/>

Parameters

Name Type Description Default Required
name String Name of the configuration directive n/a Yes
value String Value of the configuration directive n/a Yes

Supported Configuration Directives

Name Description
loglevel Log verbosity for this project
timezone Timezone for this project (dates will be shown accordingly)

 

<property/>

The Property Task lets you define variables which you can use later in your build tasks.

...
    <property name="dir" value="/var"/>
    <property name="www-dir" value="${dir}/www"/>
...

Parent Tags

  • <project/>

Parameters

Name Type Description Default Required
name String Name of the property n/a Yes
value String Value of the property n/a Yes
file String Filename of the property file to load n/a No
if String Name of property to check for boolean == true. If this property name is true, the value will be set. n/a No

 

<schedule/>

The Schedule Task sets the interval in which repository changes are looked up by the <modificationset/>.

<project name="ProjectName">
    <schedule interval="300"/>
...
</project>

Parent Tags

  • <project/>

Parameters

Name Type Description Default Required
interval Integer Number of seconds in which modifications should be detected n/a Yes

 

<cron/>

The Cron Task lets you specifiy a timer for monitoring repository by the <modificationset/>.

<project name="ProjectName">
    <cron timer="*/5 * * * *"/>
...
</project>

Parent Tags

  • <project/>

Parameters

Name Type Description Default Required
timer String Cron like configuration of timer n/a Yes

 

<modificationset/>

The Modificationset contains a set of listeners, checking for changes which can trigger a build.

<project name="ProjectName">
    <modificationset>
        <buildalways/>
    </modificationset>
...
</project>

Parent Tags

  • <project/>

Parameters

None

Supported Nested Tags

  • See Appendix C: ModificationSet

 

<builders/>

The Builders Task contains a set of one to many builders which will be executed.

<project name="ProjectName">
...
    <builders>
        <phingbuilder buildfile="build.xml" target="main"/>
    </builders>
...
</project>

Parent Tags

  • <project/>

Parameters

None

Supported Nested Tags

  • See Appendix D: Builders

 

<publishers/>

The Publishers Task contains a set of one to many publishers which will be executed after the build.

<project name="ProjectName">
...
    <publishers>
        <phingpublisher buildfile="build.xml" target="main"/>
    </publishers>
...
</project>

Parent Tags

  • <project/>

Parameters

None

Supported Nested Tags

  • See Appendix E: Publishers
